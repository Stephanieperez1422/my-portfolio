🐛 Assignment 3 — Debugging & Pseudocoding ▾

Pseudocode is plain English logic written in a structured way — not real code. The goal was to read each script, identify the bugs, fix them, and draw a corrected flowchart showing the order of steps.

Flowchart shapes
Start/Stop
Oval — begins & ends every program
Input/Output
Parallelogram — data going in or out
Process
Rectangle — a calculation or action
DEBUG01-01 — Sale Price (10% off)3 bugs
Bug 1 Used price as the variable name but it was declared as origPrice — the program can't find a variable that doesn't exist.
Bug 2 Multiplied by 0.25 which is 25% — the problem says 10% off, so it should be 0.10.
Bug 3 Typed discnt instead of discount — a typo means the variable is never found and the subtraction breaks.
discount = origPrice × 0.10
finalPrice = origPrice − discount
output finalPrice

DEBUG01-02 — Miles Per Gallon4 bugs
Bug 1 Used + (addition) instead of ÷ — miles per gallon is always miles divided by gallons, not added together.
Bug 2 Wrote gallonsOfGas — the variable declared at the top was gallonsOfGasUsed, so the calculation couldn't find it.
Bug 3 Output used milsperGallon — a typo of milesPerGallon, so nothing would actually display.
Bug 4 Last line said start again — every program must end with stop, not restart.
